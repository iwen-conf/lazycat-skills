# Lazycat Cash Incentive Quick Reference

Compiled from official pages and related docs to help move projects toward a "high-probability path for current incentives." Final reward issuance and amounts are subject to official review and current rules.

Official Sources:
- Community Incentive Rules: <https://developer.lazycat.cloud/store-rule.html>
- OIDC Integration: <https://developer.lazycat.cloud/advanced-oidc.html>
- File Association: <https://developer.lazycat.cloud/advanced-mime.html>
- Store Submission Guide: <https://developer.lazycat.cloud/store-submission-guide.html>

## 1. Reward Structure

- High-quality Self-hosted App Porting: Typically 100 RMB.
- Major Game Server Porting: Typically 300 RMB; Standard quality game servers: 100 RMB.
- Original Apps: Typically 150 to 500 RMB, evaluated based on features, UI, and stability.
- Original apps that continue to update based on user feedback have room for further incentives.
- Ported apps that integrate with the OS account system or the File Manager context menu have room for additional rewards.
- High-quality app guides: Additional reward potential.

## 2. Optimal Paths

Recommended priority:
1. Original apps with real-world scenarios.
2. High-quality self-hosted app ports with OIDC or file association integration.
3. For utility apps, prioritize file association with the network drive.
4. If the app has a login system, evaluate Lazycat OIDC integration.

## 3. Risks to Avoid

- Pure web games.
- Pure book/reader pages.
- Pure tutorial websites.
- Web-based offline apps.
- Different mods for the same game server.
- Pure database software.

These types may be allowed in the store but are not eligible for cash rewards.

## 4. Credential Rules

If an app requires a username and password:
- Regular users must be able to obtain credentials easily.
- The safest approach is to support self-registration.
- The second best is to support unified account login (OIDC).
- The minimum requirement is to provide public test credentials for regular users.

Distinguish between credential scopes:
- `lazycat_account` / `lazycat_password`: For entering Lazycat OS and opening its apps.
- `lazycat_developer_center_account` / `lazycat_developer_center_password`: For accessing the Developer Center.
- App-level accounts: Used only for internal app login, e.g., `lazycat_gitea_account` / `lazycat_gitea_password`.

If users cannot obtain credentials within the store context, it will affect submission approval.

## 5. OIDC Integration Opportunities

For apps suitable for unified accounts, prioritize OIDC:
- Set `application.oidc_redirect_path` in the manifest.
- Pass OIDC environment variables generated by the system.
- Map OIDC login to the app's internal user or session system.

This improves the login experience and favors the incentive path.

## 6. File Association Opportunities

Utility apps should prioritize `file_handler`:
- Declare supported `mime` types in `application.file_handler`.
- Configure `actions.open`.
- Implement `/open` or equivalent route handling for file parameters.

The submission guide requires utility apps to associate with corresponding file types in Lazycat Drive whenever possible.

## 7. Recommended Evidence for Incentives

- Original vs. Ported distinction.
- Upstream URL (for ports).
- Explanation of real features and use cases.
- Login/Registration/Credential acquisition instructions.
- OIDC integration details.
- File association details.
- Stability and actual testing records.

Note: OIDC is a bonus, not a guarantee for incentives. If the app's capability is weak, the experience is poor, or if the app was not verified within Lazycat OS, incentives may still be denied.

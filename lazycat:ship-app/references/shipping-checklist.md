# Lazycat 应用上架检查表

按阶段勾选。不要跳着做。

## 1. 立项与前置条件

- [ ] 明确是首发、更新，还是打回重提
- [ ] 明确目标版本号、发布时间和最小可发布范围
- [ ] 确认开发者认证、开发者中心权限、目标应用权限
- [ ] 找到仓库、构建命令、发布入口和版本来源
- [ ] 确认是否需要 `.lpk`、`lzc-cli project publish`，或两者都要

## 2. 本地应用信息

- [ ] 应用名与最终商店展示名一致
- [ ] 应用简介与真实功能一致
- [ ] icon 完整、清晰、无占位图
- [ ] 类目设置正确
- [ ] 多语言文案已覆盖默认语言，且回退策略明确
- [ ] 版本描述或 changelog 已准备

## 3. 包与上传

- [ ] 使用真实构建链路生成发布产物
- [ ] 记录包文件路径、文件名、大小、版本号
- [ ] 记录发布命令、CLI 输出或上传页面状态
- [ ] 上传前确认开发者中心当前字段和格式要求
- [ ] 如有 `.lpk`，确认包和商店资料中的版本一致

## 4. 商店资料与截图

- [ ] 手填字段与本地自动带出字段没有冲突
- [ ] 应用简介不含夸大宣传或无法证实表述
- [ ] 截图来自真实运行版本
- [ ] 截图覆盖首页、核心流程、关键差异化能力
- [ ] 截图中无调试信息、敏感数据、占位图、错误提示
- [ ] 已记录实际截图张数、尺寸或宽高比要求

## 5. 提审前测试

- [ ] 新装可启动
- [ ] 升级可完成
- [ ] 核心功能可走通
- [ ] 未登录 / 网络异常 / 空数据等失败路径可预期
- [ ] 权限、网络、目录挂载等依赖正常
- [ ] 商店信息和运行实物一致

## 6. 提审

- [ ] 整理版本号、changelog、应用简介、截图、产物信息
- [ ] 整理 reviewer 复现路径、测试账号或初始化步骤
- [ ] 提交时间、状态、审核意见已记录
- [ ] 提审证据可回溯到具体版本和构建来源

## 7. 发布后核验

- [ ] 商店可搜索或可见
- [ ] 名称、图标、简介、截图、版本号正确
- [ ] 从商店安装后的版本可正常运行
- [ ] 若商店未更新，已比对发布时间、版本号和上传记录
